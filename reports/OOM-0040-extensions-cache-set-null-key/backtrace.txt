## Why key == NULL (frame absent, already returned)
# hashtable_key_from_2_strings() (import.c ~1280) does `key = PyMem_RawMalloc(size); if (key==NULL){PyErr_NoMemory(); return NULL;}`
# _extensions_cache_find_unlocked() (import.c:1383) does `key = hashtable_key_from_2_strings(...); if (key==NULL) return NULL;`
#   -> returns at import.c:1390-1392 BEFORE `*p_key = key` (import.c:1396), so the caller's key stays NULL.
# _extensions_cache_set() (import.c:1426) `void *key = NULL;` ... entry = find_unlocked(path,name,&key);
#   entry==NULL (read as "not cached") -> _Py_hashtable_set(..., key==NULL, ...) at import.c:1496-1497 -> SEGV.

## Trigger condition
# Needs a BOUNDED OOM window (fusil --oom-seq, window>0): the key PyMem_RawMalloc fails, then the later
# alloc_extensions_cache_value() for `newvalue` SUCCEEDS, so control reaches _Py_hashtable_set. Under a
# fail-forever sweep the newvalue alloc also fails -> `goto finally` before the bad set -> no crash.
