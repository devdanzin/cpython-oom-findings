## Evidence / invariant
# exc is a valid exception instance (PyTraceBack_Here's own `assert(PyExceptionInstance_Check(exc))` passes),
# but tb_next == exc->traceback is neither NULL nor a PyTracebackObject -> PyTraceBack_Check (Py_IS_TYPE vs
# PyTraceBack_Type) fails. __traceback__ cannot be set to a non-traceback from Python (descriptor type-checks),
# so this is C-level corruption: the traceback was decref'd to zero while exc still referenced it (over-decref
# under OOM), freed, and its storage reused for an object of another type.

## To pin the producer (open)
# gdb against the vehicle: break on the raising point, then `watch -l <exc>->traceback` and continue to catch
# the decref that drops it to zero while still referenced. Likely a face of an eval-loop over-decref (cf. OOM-0005).
